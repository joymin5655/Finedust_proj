AirLens Research & Methodology

A deep dive into the technical foundations of the AirLens project, detailing our innovative CNN-LSTM architecture for air quality analysis, triple-verification methodology, and data fusion techniques using historical data.

Problem Statement

Existing air quality monitoring systems often suffer from sparse data coverage and a lack of real-time predictive capabilities. This gap leads to delayed public health warnings and an incomplete understanding of pollution dynamics. AirLens addresses this challenge by fusing satellite imagery, ground-level sensor data, and public policy information to provide accurate, hyper-local PM2.5 predictions directly in the browser, empowering individuals and researchers with actionable, real-time insights.

Methodology

CNN-LSTM Architecture

Our model employs a hybrid Convolutional Neural Network (CNN) and Long Short-Term Memory (LSTM) architecture. The CNN layers are designed to extract spatial features from satellite imagery, identifying patterns like cloud cover and aerosol density. The LSTM layers then process this spatial data alongside time-series information from ground sensors and policy data, enabling the model to learn temporal dependencies and make accurate future predictions.

Multimodal Data Fusion Architecture

Building on recent research in multimodal air quality prediction (Rowley & KarakuÅŸ 2023, AirFusion 2025), we implemented a Late Fusion architecture that combines three data modalities: image features extracted via CNN (40% weight), satellite remote sensing data (35% weight), and ground station measurements (25% weight). This approach leverages NASA MODIS for Aerosol Optical Depth (AOD), ESA Sentinel-5P for atmospheric composition (NOâ‚‚, CO, aerosol index), and OpenAQ for real-time ground truth within a 25km radius.

Key Features:
  • Multi-source data integration for enhanced accuracy
  • Weighted fusion based on data reliability and availability
  • Cross-validation with ground stations when available
  • Global coverage via satellite data APIs

Triple Verification Pipeline

To ensure the reliability of our predictions, we developed a triple verification pipeline. Data is first cross-validated against ground-truth sensor readings. Second, it is compared with outputs from established atmospheric models like CAMS. Finally, historical predictions are checked against satellite-derived aerosol optical depth (AOD) measurements, creating a robust, multi-faceted validation process that builds confidence in the model's output.

Key Innovations

  • Satellite data integration: AOD and atmospheric composition data from MODIS and Sentinel-5P
  • Adaptive weighting: Confidence scores adjust based on data availability (ground stations > satellite > image-only)
  • Cross-validation transparency: Users can see prediction errors when ground truth is available
  • Browser-based processing: All analysis runs client-side using TensorFlow.js for privacy and speed
